# Проект DDD To-Do

## Обзор

Этот проект представляет собой микросервисное приложение, построенное на основе принципов Domain-Driven Design (DDD), состоящее из двух основных сервисов:

- **ToDo Service**: API для управления задачами
- **UrlShortner Service**: Сервис сокращения URL-адресов

## Структура проекта

```
ddd-To-Do/
├── ToDo/                 # Сервис управления задачами
│   ├── src/
│   │   ├── api/         # HTTP обработчики и схемы
│   │   ├── application/ # Команды, запросы, медиатор
│   │   ├── core/        # Доменные сущности и исключения
│   │   └── infra/       # База данных и репозитории
│   ├── main.py
│   └── Dockerfile
└── UrlShortner/          # Сервис сокращения URL
    ├── src/
    │   ├── api/         # HTTP обработчики и схемы
    │   ├── db/          # Конфигурация базы данных
    │   ├── requests/    # Сущности и запросы
    │   └── services/    # Бизнес-логика
    ├── main.py
    └── Dockerfile
```

## Архитектура

Оба сервиса следуют **принципам DDD** с:
- **Слой API**: Обработка запросов и ответов
- **Слой приложения**: Команды, запросы и паттерн медиатора
- **Доменный слой**: Основная бизнес-логика и сущности
- **Слой инфраструктуры**: Реализация базы данных и репозиториев

## Начало работы

1. Соберите и запустите сервисы с Docker:
   ```bash
   docker build -t todo-service ./ToDo
   docker build -t url-shortner ./UrlShortner
   ```

2. Запустите сервисы:
   ```bash
   docker run -p 8000:8000 todo-service
   docker run -p 8001:8000 url-shortner
   ```
